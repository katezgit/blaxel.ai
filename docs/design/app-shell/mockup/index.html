Design rationale

① Notifications bell
NEW · async signal that affects Alex's flow — incident fired, batch job finished, quota threshold reached, key expiring. High value, low pixel cost. Count badge shows unread.
② Help / shortcuts (?)
NEW · keyboard shortcut sheet + docs jumps. Replaces an entire "where do I find docs" mental search with one affordance.
③ Avatar — focused menu (5 items + theme sub-menu)
Header (email + tier badge), Profile, Theme ●Dark ▸ (with Light/Dark/System sub-menu — 2-click max to toggle), Account settings →, Help & shortcuts, Log out. Deep-link opens /settings sub-app.
④ Page actions — right of title row
[Open Playground] [Rotate key] [⋯ More] labeled buttons sit on the right edge of the title row, inline with the state pill. Visual separation from topbar chrome: labeled buttons (typographic) vs small icons (iconic). Different bands, no competition.
Workspace settings — single home
Lives ONLY in the top-left workspace switcher dropdown (Switch · Create · Workspace settings →). Not duplicated in the avatar menu — same destination shouldn't have two entries. Avatar menu is for Personal + Account scopes only.
Sidebar — nav only
Three groups (Sandboxes / Hosting / Security), nothing else. No Account zone, no footer settings link, no utility chrome. Per app-shell-layout.md §2: sidebar is for navigation, topbar is for utility.
⑤ Responsive — 4 breakpoints
Wide ≥1280px: sidebar 248px full labels · search ~360px · topbar 64px.
Compact 1024–1279px: sidebar 220px · search 320px · still full shell.
Tablet 768–1023px: sidebar 64px icon rail (labels hidden, group dividers replace section headers) · ☰ hamburger appears in topbar-left · drawer opens on click for full labels · workspace stays visible.
Mobile <768px: sidebar hidden · topbar wraps to 3 rows (brand+identity / workspace / search) · page actions stack vertically with primary first. Full spec in wireframes/app-shell/index.md.
Hard rule across breakpoints
Workspace context is never hidden. The workspace switcher remains visible at every breakpoint — workspace is the operational boundary of Blaxel, not navigation decoration.
Pattern: chrome stays lean and pays pixels for high-frequency signals (notifications, help, search). Low-frequency surfaces (settings catalogs, billing, compliance) live in a dedicated /settings sub-app accessible via deep-link. Matches Datadog · Linear · W&B · Grafana · Vercel · Stripe.
